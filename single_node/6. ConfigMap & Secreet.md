## ðŸ§± ConfigMap & Secret

ðŸ§© **ConfigMap**

Digunakan untuk menyimpan **konfigurasi non-rahasia** (misal URL, nama environment, port, dsb).

Contoh:
```
APP_ENV=production
APP_NAME=MyWebApp
```
---

Data ini bisa di-inject ke Pod sebagai environment variable atau file konfigurasi.

---

ðŸ”’ **Secret**

Digunakan untuk menyimpan data sensitif seperti password, API key, token, dsb.
Secara default, Secret dienkode dalam base64 (bukan enkripsi penuh, tapi tetap lebih aman dari ConfigMap).

Contoh:
```
DB_USER=admin
DB_PASS=SuperSecret123
```

## ðŸ“„ Membuat Config Map & Secreet

ðŸ§© **ConfigMap**
```yaml
apiVersion: v1                        # Versi API untuk ConfigMap
kind: ConfigMap                       # Jenis resource
metadata:
  name: app-config                    # Nama ConfigMap
data:                                 # Data non-sensitif dalam bentuk key:value
  APP_ENV: "production"
  APP_NAME: "MyWebApp"
```

ðŸ”’ **Secret**

```yaml
apiVersion: v1                        # Versi API untuk Secret
kind: Secret                          # Jenis resource
metadata:
  name: db-secret                     # Nama Secret
type: Opaque                          # Tipe secret umum (custom key:value)
data:                                 # Data sensitif dalam base64
  DB_USER: YWRtaW4=                   # "admin" dalam base64
  DB_PASS: U3VwZXJTZWNyZXQxMjM=       # "SuperSecret123" dalam base64
```

## ðŸ§© Menggunakan ConfigMap & Secret di Pod / Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp
spec:
  replicas: 1
  selector:
    matchLabels:
      app: webapp
  template:
    metadata:
      labels:
        app: webapp
    spec:
      containers:
      - name: nginx
        image: nginx:alpine
        envFrom:
        - configMapRef:
            name: app-config           # Ambil semua key:value dari ConfigMap
        - secretRef:
            name: db-secret            # Ambil semua key:value dari Secret
```

## ðŸ§¹ Menghapus & Mengedit

kubectl edit secret db-secret
