**Pod** adalah unit terkecil di Kubernetes.
Ia membungkus satu atau lebih container yang berjalan di satu node yang sama.

---

Biasanya 1 Pod = 1 container (misal 1 Nginx, 1 Redis, dll).
Pod bersifat **sementara** â€” kalau Pod mati, Kubernetes bisa membuat ulang (melalui Deployment atau ReplicaSet).

## âš™ï¸ Membuat Pod
Terdapat dua cara untuk membuat pod yaitu imperatif(command langsung), dan deklaratif(file .yaml)

### Imperatif

---

kubectl run <nama_pod> --image=<nama_image:tag> --port=<port_expose_pod>

cth:

kubectl run mynginx --image=nginx:alpine --port=80

---

### Deklaratif
1. Buat file .yaml :

```yaml
apiVersion: v1                     # Versi API untuk resource Pod
kind: Pod                          # Jenis resource yang dibuat adalah Pod
metadata:
  name: mynginx                    # Nama Pod
spec:
  containers:                      # Daftar container yang dijalankan dalam Pod
  - name: nginx                    # Nama container di dalam Pod
    image: nginx:alpine            # Image Docker yang digunakan (diambil dari Docker Hub)
    ports:
    - containerPort: 80            # Port aplikasi di dalam container (bukan port publik)


```
2. Jalankan perintah
```
kubectl apply -f <nama_file>.yaml
cth:
kubectl apply -f mynginx.yaml
```

## ğŸ” Cek Pod

1. Melihat semua pod
```
kubectl get pods
```
2. Melihat detail lebih lengkap:
```
kubectl get pods -o wide
```
3. Melihat deskripsi dari pod:
```
kubectl describe pod <nama_pod>
```
4. Melihat log:
```
kubectl logs <nama_pod>
```
5. Masuk kedalam pod dengan mode interaktif:
```
kubectl exec -it <nama_pod> --sh
```

## ğŸ” Mematikan/delete pod

1. Jika pod dijalankan dengan cara imperatif, maka untuk delete dapat menggunakan:
```
kubectl delete pod <nama_pod>
```
2. Jika pod dijalankan dengan cara deklaratif, maka delete dapat menggunakan:
```
kubectl delete -f <nama_file>.yaml
```
